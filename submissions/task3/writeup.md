# Task 3 Writeup

`configs/v6.yaml` is not a new base model; it is a smarter configuration around the same Nemotron model. The meaningful change is that it combines the stronger `v3_prompt_hardening.txt` main prompt with the `v5` sandwich guardrail shape: input classifier, main assistant, then output validator. The production hypothesis was that this would keep valid travel answers working while reducing jailbreak/off-topic leakage, because suspicious input is blocked early and any leaked output is checked again before returning to the user. The latest comparable full eval supports that hypothesis: latest `v5` had `accuracy_overall=0.970`, `accuracy_jailbreak=0.920`, `verdict_rate_leaked=0.020`, `avg_latency_seconds=1.164`, `p95=4.385`, and `total_cost_usd=$0.0245`; promoted `v6` version 5 had `accuracy_overall=0.990`, `accuracy_jailbreak=0.960`, `verdict_rate_leaked=0.000`, `avg_latency_seconds=0.850`, `p95=3.148`, and `total_cost_usd=$0.0214`. So the evidence says `v6` was more precise, less prone to jailbreak leakage, faster on average and p95, and cheaper on this eval run. The Task 2 CLI confirms that the `production` alias now points to `config_id=v6`, MLflow version 5. Part D is also complete: `POST /admin/reload` returned `status=ok`, and Prometheus `assistant_info` shows `config_id="v6"`, `model_alias="production"`, and `model_version="5"` for the live service.
